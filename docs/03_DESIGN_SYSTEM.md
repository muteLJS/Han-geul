# 디자인 시스템

## 최상위 디자인 원칙
- 현재 제품은 모바일 웹 버전이다.
- 모든 화면은 `max-w-lg` 모바일 앱 셸 안에서 통일한다.
- 화면 배경은 한지 질감만 사용한다.
- 흰색/옅은 한지색 표면은 카드, 입력창, 말풍선 같은 보조 UI에만 사용한다.
- 데스크탑 전용 레이아웃을 따로 만들지 않는다.

| ink | #1A1714 | 본문 텍스트 |

| ob-white | #F5F0E8 | 어휘 연습 오방색 |
| ob-red | #C4614A | 필사 오방색 |
| ob-black | #3A3530 | 나를 담은 글 오방색 |
| ob-yellow | #D4A853 | 나의 책 쓰기 오방색, 보상 |
| tg-blue | #003478 | 주요 CTA, 링크, 강조 |
| tg-red | #CD2E3A | 한이 꼭대기 점, 경고 |

## 서체
| 영역 | 서체 | 이유 |
|---|---|---|
| 제목 | Noto Serif KR | 전통적이지만 단정한 인상 |
| 본문/UI | Noto Sans KR | 모바일 작은 크기 판독성 |
| 한이 말풍선 | Nanum Pen Script | 손글씨 같은 따뜻함 |

## 배경 규칙
- 화면 단위 배경 클래스는 `hanji-bg`만 사용한다.
- `bg-stone-*`, 무채색 전체 배경, 그라디언트 전체 배경은 사용하지 않는다.
- 페이지 내부에서 `bg-[#FAF6EE]`로 한지 질감을 덮지 않는다.
- 공통 레이아웃: `PageLayout`, `(auth)/layout.js`, `(onboarding)/layout.js`가 한지 배경을 책임진다.

## 오방색 버튼 시스템
- 배치: 홈에서 원라인 5개
- 크기: 모바일 터치 기준 정사각형 버튼
- 순서: 청(자유 글쓰기) → 백(어휘) → 적(필사) → 흑(나를 담은 글) → 황(책 쓰기)

## 한이 크기 기준
| 상황 | 크기 |
|---|---|
| 스플래시 | 64×90px 기준 |
| 온보딩/완료 | 120px 이상 |
| 홈 메인 | 64px 이상 |
| 글쓰기 플로팅 | 64px |
| 모달 내부 | 32px 이상 |

## 디자인 방향
- 한지 질감, 먹색, 절제된 오방색, 여백, 선 중심
- 경복궁 같은 절제된 전통미 + 현대적 모바일 UI
- 기능보다 먼저 화면 간 배경, 간격, 버튼 톤을 통일한다.
