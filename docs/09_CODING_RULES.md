# 코딩 규칙 (반드시 준수)

## 절대 규칙
1. 이 문서와 제품 문서에 없는 기능을 임의로 추가하지 않는다
2. 마스코트 이름은 반드시 "한이"다 — "나래" 사용 절대 금지
3. 페르소나 1은 정근우(28세, 직장인)다
4. 페르소나 2는 이젠(34세, 프리랜서)다
5. 현재 목표 플랫폼은 모바일 웹이다
6. 전체 화면 배경은 한지 질감(`hanji-bg`)만 사용한다

## 모바일 웹 UI 규칙
7. 모든 사용자 화면은 `max-w-lg` 모바일 앱 셸 안에서 구현한다
8. 데스크탑 전용 그리드/상단 네비를 먼저 만들지 않는다
9. 페이지 최상위 배경을 `bg-white`, `bg-stone-*`, 임의 그라디언트로 덮지 않는다
10. 흰색/옅은 한지색은 카드, 입력창, 말풍선 같은 보조 표면에만 사용한다

## 홈 화면 규칙
11. 홈에서 보유 포인트 카드를 노출하지 않는다
12. 홈에서 도서함 카드를 노출하지 않는다
13. 알림 아이콘을 사용하지 않는다
14. 오방색 버튼은 원라인 5개 배치다

## 탐색 규칙
15. "인기순" 표현을 사용하지 않는다 → "조회순" 사용
16. 팔로우 수 숫자를 노출하지 않는다
17. 조회수 숫자를 노출하지 않는다
18. 탐색은 책과 작가 발견 중심이다 (글 피드 중심 금지)

## 글쓰기 규칙
19. 작성 중 글자 수 카운터를 노출하지 않는다
20. 저장 버튼은 항상 노출한다 (내용 없으면 비활성)
21. 글자 수는 저장 후 또는 성장 리포트에서만 표시한다

## AI 피드백 규칙
22. 금지 표현을 절대 사용하지 않는다
    - 금지: 틀렸어요, 부족해요, 어색해요, 점수는 N점입니다
23. 피드백은 페이지 이동 없이 모달/토스트로 제공한다
24. 무료 회원 하루 3회 제한을 서버에서 검증한다

## 구현 순서
1. 진입 흐름: 스플래시, 온보딩, 로그인, 회원가입
2. 홈 화면
3. 글쓰기 에디터 + AI 피드백 모달
4. 도서함 책장 UI
5. 어휘/필사/나를 담은 글
6. 나의 책 쓰기
7. 탐색 + 작가 채널
8. 구독 페이지 + 모달
9. 한이 꾸미기
10. 마이페이지 + 설정

## 네이밍 규칙
- 컴포넌트: PascalCase (예: BookSpine.js)
- 훅: camelCase, use 접두사 (예: useAutoSave.js)
- API 라우트: kebab-case (예: /api/ai/feedback)
- 상수: UPPER_SNAKE_CASE (예: MAX_FREE_FEEDBACK)
- CSS 클래스: Tailwind 유틸리티 클래스 우선

## 보안 규칙
- .env.local을 GitHub에 절대 업로드하지 않는다
- 공개/비공개 권한은 프론트뿐 아니라 서버에서도 검증한다
- AI API 호출에 Rate Limit을 적용한다
- 사용자 입력값은 서버에서 sanitize한다
