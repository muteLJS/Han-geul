# 코딩 규칙 (반드시 준수)

## 절대 규칙
1. 이 문서에 없는 기능을 임의로 추가하지 않는다
2. 마스코트 이름은 반드시 "한이"다 — "나래" 사용 절대 금지
3. 페르소나 1은 정근우(28세, 직장인)다
4. 페르소나 2는 이젠(34세, 프리랜서)다

## 홈 화면 규칙
5. 홈에서 보유 포인트 카드를 노출하지 않는다
6. 홈에서 도서함 카드를 노출하지 않는다
7. 알림 아이콘을 사용하지 않는다
8. 오방색 버튼은 원라인 5개 배치다

## 탐색 규칙
9. "인기순" 표현을 사용하지 않는다 → "조회순" 사용
10. 팔로우 수 숫자를 노출하지 않는다
11. 조회수 숫자를 노출하지 않는다
12. 탐색은 책과 작가 발견 중심이다 (글 피드 중심 금지)

## 글쓰기 규칙
13. 작성 중 글자 수 카운터를 노출하지 않는다
14. 저장 버튼은 항상 노출한다 (내용 없으면 비활성)
15. 글자 수는 저장 후 또는 성장 리포트에서만 표시한다

## AI 피드백 규칙
16. 금지 표현을 절대 사용하지 않는다
    금지: 틀렸어요, 부족해요, 어색해요, 점수는 N점입니다
17. 피드백은 페이지 이동 없이 모달/토스트로 제공한다
18. 무료 회원 하루 3회 제한을 서버에서 검증한다

## 구현 순서 (반드시 지킨다)
1. 홈 화면
2. 글쓰기 에디터 + AI 피드백 모달
3. 로그인 + 비회원 처리
4. 도서함 책장 UI
5. 온보딩
6. 어휘/필사/나를 담은 글
7. 나의 책 쓰기
8. 탐색 + 작가 채널
9. 구독 페이지 + 모달
10. 한이 꾸미기
11. 마이페이지 + 설정

## 네이밍 규칙
- 컴포넌트: PascalCase (예: BookSpine.js)
- 훅: camelCase, use 접두사 (예: useAutoSave.js)
- API 라우트: kebab-case (예: /api/ai/feedback)
- 상수: UPPER_SNAKE_CASE (예: MAX_FREE_FEEDBACK)
- CSS 클래스: Tailwind 유틸리티 클래스 우선

## 보안 규칙
- .env.local을 GitHub에 절대 업로드하지 않는다
- 공개/비공개 권한은 프론트뿐 아니라 서버에서도 검증한다
- AI API 호출에 Rate Limit을 적용한다
- 사용자 입력값은 서버에서 sanitize한다
