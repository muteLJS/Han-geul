# 프론트 페이지 목록 및 라우트

## 모바일 웹 셸 원칙
- 모든 사용자 화면은 모바일 웹 기준으로 설계한다.
- 앱 셸 최대 너비는 `max-w-lg`이며 데스크탑에서도 중앙 정렬된 모바일 화면을 유지한다.
- 화면 배경은 한지 질감(`hanji-bg`)만 사용한다.
- 카드, 입력창, 버튼은 한지 배경 위에 얹는 보조 표면으로만 사용한다.

## 공통 모달 컴포넌트
| 컴포넌트 | 노출 시점 |
|---|---|
| AuthModal | 비회원이 저장/반응/팔로우 시도 시 |
| SubscribeModal | 구독 제한 기능 접근 시 |
| FeedbackModal | 피드백 받기 클릭 시 |
| SaveModal | 저장 버튼 클릭 시 |
| PublishModal | 공개 범위 설정 시 |

## 우선 구현 순서
1. 진입 흐름: 스플래시, 온보딩, 로그인, 회원가입
2. 홈
3. 글쓰기 에디터 + AI 피드백 모달
4. 도서함 책장 UI
5. 어휘/필사/나를 담은 글
6. 나의 책 쓰기
7. 탐색 + 작가 채널
8. 구독 페이지 + 모달
9. 한이 꾸미기
10. 마이페이지 + 설정
